## 2. テスト概要

### 2.1 テスト目的

基本設計書に定義された各機能（FNC-001〜FNC-009）が正しく動作することを、単体レベルで検証する。

### 2.2 テスト範囲

| 対象 | テスト内容 |
|------|----------|
| server/models/user.js | ユーザー作成・認証ロジック |
| server/models/record.js | 履歴CRUD操作 |
| server/routes/auth.js | 認証API（ログイン・ログアウト・登録） |
| server/routes/records.js | 履歴API（登録・取得・削除・グラフデータ） |

### 2.3 テスト環境

| 項目 | 内容 |
|------|------|
| テストフレームワーク | Jest |
| HTTPテスト | supertest |
| データベース | SQLite（テスト用インメモリまたは一時ファイル） |
| 実行コマンド | `npm test` |

## 3. テストケース一覧

### 3.1 ユーザーモデル（server/models/user.js）

| テストID | テスト項目 | テスト内容 | 期待結果 |
|----------|-----------|----------|---------|
| UT-M-001 | ユーザー作成 | 正常なユーザー名とパスワードでユーザーを作成 | ユーザーIDが返却される |
| UT-M-002 | ユーザー認証成功 | 正しいユーザー名・パスワードで認証 | ユーザー情報（id, username）が返却される |
| UT-M-003 | ユーザー認証失敗（パスワード不一致） | 誤ったパスワードで認証 | nullが返却される |
| UT-M-004 | ユーザー認証失敗（存在しないユーザー） | 存在しないユーザー名で認証 | nullが返却される |
| UT-M-005 | ユーザー検索 | 存在するユーザー名で検索 | ユーザー情報が返却される |
| UT-M-006 | ユーザー検索（存在しない） | 存在しないユーザー名で検索 | undefinedが返却される |

### 3.2 履歴モデル（server/models/record.js）

| テストID | テスト項目 | テスト内容 | 期待結果 |
|----------|-----------|----------|---------|
| UT-M-101 | 履歴登録 | 日付・身長・体重・BMIを登録 | レコードIDが返却される |
| UT-M-102 | 履歴更新（UPSERT） | 同一ユーザー・同一日付で再登録 | 既存レコードが更新される |
| UT-M-103 | 履歴一覧取得 | ユーザーの全履歴を取得 | 日付降順でレコードが返却される |
| UT-M-104 | 履歴削除 | 指定IDのレコードを削除 | trueが返却され、レコードが削除される |
| UT-M-105 | 履歴削除（他ユーザー） | 他ユーザーのレコードを削除 | falseが返却され、削除されない |
| UT-M-106 | グラフ用データ取得 | 過去6か月間のデータを取得 | 日付昇順でレコードが返却される |
| UT-M-107 | グラフ用データ取得（期間外） | 6か月以上前のデータが除外される | 期間外のレコードが含まれない |
