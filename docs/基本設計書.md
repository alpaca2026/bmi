## 2. システム概要

### 2.1 システムの目的

体重と身長を入力してBMIを算出・表示するWebアプリケーションを構築する。ユーザー認証、履歴保存、推移グラフ表示の機能を備える。

### 2.2 システム構成

本アプリケーションはクライアント・サーバー構成とする。フロントエンドはHTML・CSS・JavaScriptで構成し、バックエンドでユーザー認証とデータ永続化を行う。

```
┌─────────────────────────────┐     ┌─────────────────────────────┐
│      フロントエンド（ブラウザ）   │     │       バックエンド（サーバー）   │
│                             │     │                             │
│  ┌───────┐  ┌───────┐      │     │  ┌───────────────────────┐  │
│  │ HTML  │  │  CSS  │      │     │  │   認証処理             │  │
│  └───────┘  └───────┘      │ HTTP│  └───────────────────────┘  │
│  ┌───────────────────────┐  │←───→│  ┌───────────────────────┐  │
│  │     JavaScript        │  │     │  │   履歴データ管理        │  │
│  │ ・画面制御             │  │     │  └───────────────────────┘  │
│  │ ・入力バリデーション    │  │     │  ┌───────────────────────┐  │
│  │ ・BMI算出ロジック      │  │     │  │   データベース          │  │
│  │ ・グラフ描画           │  │     │  │  ・ユーザー情報         │  │
│  └───────────────────────┘  │     │  │  ・BMI履歴データ        │  │
└─────────────────────────────┘     │  └───────────────────────┘  │
                                    └─────────────────────────────┘
```

### 2.3 技術スタック

| 項目 | 技術 |
|------|------|
| フロントエンド | HTML5 / CSS3 / JavaScript（ES6+） |
| グラフ描画 | Chart.js |
| バックエンド | Node.js / Express（または同等のWebフレームワーク） |
| データベース | SQLite（または同等のRDB） |
| 認証方式 | セッションベース認証 |
| 動作環境 | モダンブラウザ（Chrome, Firefox, Safari, Edge） |

## 4. 機能設計

### 4.1 機能一覧

| 機能ID | 機能名 | 概要 |
|--------|--------|------|
| FNC-001 | ログイン | ユーザー名・パスワードで認証を行う |
| FNC-002 | ログアウト | セッションを破棄しログイン画面に戻る |
| FNC-003 | 入力バリデーション | 各入力値の妥当性を検証する |
| FNC-004 | BMI算出 | 身長と体重からBMI値を算出する |
| FNC-005 | BMI判定 | BMI値から肥満度判定を行う |
| FNC-006 | 履歴保存 | BMI算出結果を日付とともに保存する |
| FNC-007 | 履歴一覧表示 | ユーザーの過去の記録を一覧表示する |
| FNC-008 | 履歴削除 | 選択した履歴レコードを削除する |
| FNC-009 | グラフ表示 | 過去6か月間の推移を折れ線グラフで表示する |

### 4.2 FNC-001: ログイン

#### 処理概要

ログインボタン押下時に、入力されたユーザー名・パスワードで認証を行う。

#### 処理仕様

- ユーザー名・パスワードをサーバーに送信し認証を行う
- 認証成功時: セッションを生成し、BMI算出画面（SCR-002）に遷移する
- 認証失敗時: エラーメッセージ「ユーザー名またはパスワードが正しくありません」を表示する
- パスワードはハッシュ化して保存・照合する

### 4.3 FNC-002: ログアウト

#### 処理概要

ログアウトボタン押下時に、セッションを破棄してログイン画面に遷移する。

#### 処理仕様

- サーバーサイドのセッションを破棄する
- ログイン画面（SCR-001）にリダイレクトする

### 4.4 FNC-003: 入力バリデーション

#### 処理概要

各画面での入力値の妥当性を検証する。

#### ログイン画面バリデーション

| 優先順 | 検証対象 | 検証内容 | エラーメッセージ |
|--------|----------|----------|------------------|
| 1 | ユーザー名 | 未入力チェック | ユーザー名を入力してください |
| 2 | パスワード | 未入力チェック | パスワードを入力してください |

#### BMI算出画面バリデーション

| 優先順 | 検証対象 | 検証内容 | エラーメッセージ |
|--------|----------|----------|------------------|
| 1 | 日付 | 未入力チェック | 日付を入力してください |
| 2 | 日付 | 日付形式チェック | 正しい日付を入力してください |
| 3 | 身長 | 未入力チェック | 身長を入力してください |
| 4 | 体重 | 未入力チェック | 体重を入力してください |
| 5 | 身長 | 数値チェック | 身長は数値で入力してください |
| 6 | 体重 | 数値チェック | 体重は数値で入力してください |
| 7 | 身長 | 正の数値チェック（> 0） | 身長は正の数値で入力してください |
| 8 | 体重 | 正の数値チェック（> 0） | 体重は正の数値で入力してください |

#### 処理仕様

- バリデーションは優先順に従い上から順に実行する
- すべてのバリデーションエラーを同時に表示する
- エラーがある場合、後続処理は実行しない

### 4.5 FNC-004: BMI算出

#### 処理概要

入力された身長と体重からBMI値を算出する。

#### 計算式

```
BMI = weight / (height / 100)²
```

| 変数 | 内容 | 単位 |
|------|------|------|
| weight | 入力された体重 | kg |
| height | 入力された身長 | cm |

#### 処理仕様

- 身長をcmからmに変換する（÷100）
- BMI値を算出する
- 小数点以下第2位を四捨五入し、第1位まで表示する

### 4.6 FNC-005: BMI判定

#### 処理概要

算出されたBMI値をもとに、日本肥満学会基準に基づく判定を行う。

#### 判定ロジック

```
if (BMI < 18.5) → "低体重（やせ）"
else if (BMI < 25.0) → "普通体重"
else if (BMI < 30.0) → "肥満（1度）"
else if (BMI < 35.0) → "肥満（2度）"
else if (BMI < 40.0) → "肥満（3度）"
else → "肥満（4度）"
```

### 4.7 FNC-006: 履歴保存

#### 処理概要

BMI算出結果を日付・ユーザーIDとともにデータベースに保存する。

#### 処理仕様

- ログイン中のユーザーIDを取得する
- 入力された日付、身長、体重、算出されたBMI値を保存する
- 同一ユーザー・同一日付のデータが既存の場合は上書き更新（UPSERT）する
- 保存成功後、履歴一覧を再読込して表示を更新する

### 4.8 FNC-007: 履歴一覧表示

#### 処理概要

ログインユーザーの過去の記録を一覧表示する。

#### 処理仕様

- ログインユーザーの全履歴をデータベースから取得する
- 日付の降順（新しい順）でテーブルに表示する
- 各行に削除ボタンを配置する

### 4.9 FNC-008: 履歴削除

#### 処理概要

選択した履歴レコードを削除する。

#### 処理仕様

- 削除ボタン押下時に確認ダイアログを表示する
- 確認後、該当レコードをデータベースから削除する
- 削除後、履歴一覧を再読込して表示を更新する

### 4.10 FNC-009: グラフ表示

#### 処理概要

過去6か月間の体重・BMI値の推移を折れ線グラフで描画する。

#### 処理仕様

- 現在日付から過去6か月間のデータをデータベースから取得する
- Chart.jsを使用して折れ線グラフを描画する
- X軸: 日付（時系列）
- Y軸（左）: 体重（kg）
- Y軸（右）: BMI値
- データポイント間は直線で接続する
- データがない日付はプロットしない

## 6. データ設計

### 6.1 テーブル定義

#### 6.1.1 ユーザーテーブル（users）

| カラム名 | データ型 | 制約 | 説明 |
|----------|----------|------|------|
| id | INTEGER | PRIMARY KEY, AUTOINCREMENT | ユーザーID |
| username | VARCHAR(50) | UNIQUE, NOT NULL | ユーザー名 |
| password_hash | VARCHAR(255) | NOT NULL | パスワードハッシュ |
| created_at | DATETIME | NOT NULL, DEFAULT CURRENT_TIMESTAMP | 作成日時 |

#### 6.1.2 BMI履歴テーブル（bmi_records）

| カラム名 | データ型 | 制約 | 説明 |
|----------|----------|------|------|
| id | INTEGER | PRIMARY KEY, AUTOINCREMENT | レコードID |
| user_id | INTEGER | FOREIGN KEY (users.id), NOT NULL | ユーザーID |
| record_date | DATE | NOT NULL | 記録日付 |
| height | DECIMAL(5,1) | NOT NULL | 身長（cm） |
| weight | DECIMAL(5,1) | NOT NULL | 体重（kg） |
| bmi | DECIMAL(4,1) | NOT NULL | BMI値 |
| created_at | DATETIME | NOT NULL, DEFAULT CURRENT_TIMESTAMP | 作成日時 |
| updated_at | DATETIME | NOT NULL, DEFAULT CURRENT_TIMESTAMP | 更新日時 |

- UNIQUE制約: (user_id, record_date) の組み合わせで一意

### 6.2 API定義

| メソッド | エンドポイント | 概要 |
|----------|---------------|------|

| GET | /api/records | 履歴一覧取得 |

## 7. エラー処理設計

### 7.1 エラー処理方針

| エラー種別 | 処理方法 |
|------------|----------|
| 入力バリデーションエラー | エラー表示エリアにメッセージを表示し、後続処理を中止する |
| 認証エラー | エラーメッセージを表示し、ログイン画面に留まる |
| セッション切れ | ログイン画面にリダイレクトする |
| サーバーエラー | ユーザーに汎用エラーメッセージを表示する |
| 通信エラー | ユーザーにネットワークエラーメッセージを表示する |

### 7.2 エラーメッセージ一覧

| エラーID | メッセージ | 発生条件 |
|----------|----------|----------|
| ERR-001 | ユーザー名を入力してください | ユーザー名が未入力 |
| ERR-002 | パスワードを入力してください | パスワードが未入力 |
| ERR-003 | ユーザー名またはパスワードが正しくありません | 認証失敗 |
| ERR-004 | 日付を入力してください | 日付が未入力 |
| ERR-005 | 正しい日付を入力してください | 日付形式が不正 |
| ERR-006 | 身長を入力してください | 身長が未入力 |
| ERR-007 | 体重を入力してください | 体重が未入力 |
| ERR-008 | 身長は数値で入力してください | 身長が数値でない |
| ERR-009 | 体重は数値で入力してください | 体重が数値でない |
| ERR-010 | 身長は正の数値で入力してください | 身長が0以下 |
| ERR-011 | 体重は正の数値で入力してください | 体重が0以下 |
| ERR-012 | セッションが切れました。再度ログインしてください | セッション期限切れ |
| ERR-013 | サーバーエラーが発生しました | サーバー内部エラー |
